#Part 1: Introduction to Software Engineering

Explain what software engineering is and discuss its importance in the technology industry.

Software Engineering is the development , maintenance and design of software . It is important to the technology industry because it allows the industry to scale , grow and enhance the offering that it has to give to the public.


Identify and describe at least three key milestones in the evolution of software engineering.

The three key milestones in the evolution of software engineering is the creation of programming languages that allows developers to continuously grow technology and contribute to society with the cutting edge technology. Software engineering became a discipline which moved the approach from a crafts approach to an engineering approach that required standards and methodologies that should be followed when engineering software.


List and briefly explain the phases of the Software Development Life Cycle.

1. Requirements : This is documenting and understanding what the user wants. 
2. Design : Creating the interface in which a user will interact with the project and creating the software architecture for the project.
3. Implementation : Writing the code and implementing it according to design specifications.
4. Testing : Conducting various test to ensure the software meets standard and works well.
5. Deployment: Releasing the software to users or customers. 
6. Maintenance : Providing continuous support , update and enhancement to software after deployment.


Compare and contrast the Waterfall and Agile methodologies. Provide examples of scenarios where each would be appropriate.

The waterfall and agile methods are different because with the waterfall method there is a rather strict steps that have to be followed for the development of the project to happen , whereas with the agile method allows room for anything to happen and to take things as they come. 

Waterfall method would make sense in smaller developing teams that don't have the abundance of individuals working on the project. 
Agile method works well with bigger teams as they would be able to tackle anything that comes in accordance to the methodology.

Describe the roles and responsibilities of a Software Developer, a Quality Assurance Engineer, and a Project Manager in a software engineering team.

A software developer , codes the software with given requirements.
Quality Assurance Engineering , ensures that the software is up to standard and requirement. 
Project manager , manages the project and individuals involved to ensure that the customers needs are met.

Discuss the importance of Integrated Development Environments (IDEs) and Version Control Systems (VCS) in the software development process. Give examples of each.

IDE , allows developers to code , test and debug what they have written such as Visual Studio COde. 
Version Control System , allows there to be a management of how the software has changed and who has done it and why. It creates accountability and transparency when collaborating on a project , or example Git.

What are some common challenges faced by software engineers? Provide strategies to overcome these challenges.

1. Changing requirements , because clients are often always unpredictable you need to ensure there is room that allow the change not to impact what you're doing in a huge way. A strategy would be to ensure the build is flexible to change

2. Tight deadlines , mean that work can often be rushed and not done concisely . To tackle this creating a trick timetable will enable the individuals to know when every element needs to be pushed out.

Explain the different types of testing (unit, integration, system, and acceptance) and their importance in software quality assurance.

1. Unit testing , is testing individual components of the build . This enables to ensure that the individual components work outside of the larger component with no problem.

2. Integration testing , is seeing if the components work well together and this is important to know that anything that you integrate into the system won't fail it and that it is compatible.

3. System testing , is testing the entire build to see if anything is wrong. 

4. Acceptance testing , is testing the software against user requirements to see if it is what the user needs.
